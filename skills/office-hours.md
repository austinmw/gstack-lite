# /office-hours

## TL;DR

### Intent

Use this before writing code when the idea is still fuzzy.

The goal is to turn a vague concept into a sharper design note by forcing clarity on the user, the pain, the wedge, and the next move.

### How to use it

Bring it an idea, early feature concept, or product direction.

Use it when you need help deciding what problem you are actually solving and what the first real version should be.

### What is actually useful here

The genuinely useful part is the pressure test:

- who exactly has the problem
- what they do today
- what evidence is real versus hypothetical
- what the smallest worthwhile wedge is
- what 2-3 plausible directions look like

That is better than generic brainstorming. Most of the original ceremony is not.

## Core Use

This is the front door skill.

Use it for:

- "I have an idea."
- "Is this worth building?"
- "Help me think through this before I start coding."

Do not use it for implementation planning. It should end in a design note, not code.

## Two Modes

### Startup mode

Use this when the project might become a company, internal product, or serious business effort.

The useful move here is to force specificity:

- real user, not market segment
- real pain, not vague interest
- current workaround, not imagined demand
- narrow wedge, not platform fantasy

### Builder mode

Use this when the project is for fun, learning, open source, demos, or exploration.

The useful move here is different:

- find the most delightful version
- make it specific enough to build
- pick the fastest path to something worth showing

## Minimal Flow

1. Figure out whether this is startup mode or builder mode.
2. Get specific about the problem and the user.
3. Challenge the current framing instead of accepting it.
4. Generate 2-3 distinct approaches.
5. Recommend one.
6. Write a short design note capturing the decision.

## Rules

- Do not start coding.
- Do not confuse politeness with demand.
- Do not let the user stay vague if specificity is the whole point.
- Do end with a recommendation and a usable design note.

## In One Sentence

`/office-hours` is useful because it makes the idea harder to fake and easier to scope.
